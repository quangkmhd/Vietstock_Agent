# Vietstock Agent Product Documentation Hub

This directory contains the formal product and technical specifications for the Vietstock Agent system. These documents serve as the single source of truth for the system's design, requirements, and operational procedures.

## Documentation Map

| Document | Description |
| :--- | :--- |
| **[PRD.md](./PRD.md)** | Product Requirements Document - Core goals and user stories. |
| **[DESIGN.md](./DESIGN.md)** | System Design - High-level architecture and agent coordination. |
| **[DATA.md](./DATA.md)** | Data Architecture - Schema definitions and storage strategy. |
| **[API.md](./API.md)** | API Specification - Tool definitions and interface schemas. |
| **[IO_DEFINITION.md](./IO_DEFINITION.md)** | I/O Definition - Standardized input/output contracts. |
| **[PROMPTS.md](./PROMPTS.md)** | Prompt Engineering - Agent personas and system instructions. |
| **[EVALUATION.md](./EVALUATION.md)** | Evaluation Framework - Metrics and testing strategies. |
| **[RUNBOOK.md](./RUNBOOK.md)** | Operations Runbook - Deployment and maintenance procedures. |
| **[TROUBLESHOOTING.md](./TROUBLESHOOTING.md)** | Troubleshooting Guide - Known issues and resolutions. |
| **[EXPERIMENTS.md](./EXPERIMENTS.md)** | Experiments Log - Findings from R&D phases. |

## Standards
- **Version Control**: All documents must include a version history.
- **AI-Readability**: Use clear, structured Markdown for both human and LLM consumption.
- **Consistency**: All architectural changes must be reflected across the PRD, Design, and Data docs.
